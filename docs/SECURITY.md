| Version | Unterstützt |

## Sicherheitslücke melden

Wenn Sie eine Sicherheitslücke in diesem Projekt entdecken, bitten wir Sie, diese **verantwortungsvoll** zu melden.

### Meldung

- **E-Mail:** Bitte senden Sie eine detaillierte Beschreibung an die im Impressum angegebene E-Mail-Adresse
- **Betreff:** `[SECURITY] Kurzbeschreibung der Sicherheitslücke`
- **Nicht:** Erstellen Sie bitte **keinen** öffentlichen GitHub Issue für Sicherheitslücken

### Was wir benötigen

- Beschreibung der Sicherheitslücke
- Schritte zur Reproduktion
- Mögliche Auswirkungen
- Optional: Vorschlag zur Behebung

### Unser Prozess

1. **Bestätigung:** Wir bestätigen den Eingang innerhalb von 48 Stunden
2. **Analyse:** Wir bewerten die Schwere und priorisieren die Behebung
3. **Behebung:** Kritische Lücken werden so schnell wie möglich gepatcht
4. **Veröffentlichung:** Nach dem Fix informieren wir über die Änderung im CHANGELOG

### Umfang

Diese Policy gilt für:

- Den Quellcode dieses Repositories
- Die unter `fiecon-consulting.eu` gehostete Website
- Die API-Endpoints (`/api/contact`, `/api/health`)

### Ausschlüsse

Folgende Punkte sind **nicht** im Umfang:

- Social Engineering (Phishing, etc.)
- Denial-of-Service-Angriffe
- Schwachstellen in Drittanbieter-Diensten
